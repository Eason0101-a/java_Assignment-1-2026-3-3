# Java 程式設計與數學模型模擬之實作

Implementation of Java Programming and Mathematical Model Simulation

## 專案簡介
這個專案是一個以 Java 撰寫的數學模型模擬器，透過 Euler 方法對常微分方程進行數值近似。程式執行於命令列介面，使用者可選擇不同模型、輸入參數，並查看每一步的模擬結果與最終數值。

目前 Main.java 支援以下兩種模型：

1. 線性衰減模型：dx/dt = -k*x
2. 非線性 Logistic 模型：dx/dt = r*x*(1 - x/K)

## 程式功能
- 提供模型選單，讓使用者在兩種微分方程模型中擇一模擬。
- 使用 Euler 方法逐步計算每個時間點的 x 值。
- 顯示 step、time、x 的表格式輸出。
- 顯示模擬結束後的最終結果。
- 具備基本輸入驗證與例外處理。

## 程式邏輯
Main.java 主要包含以下結構：

1. computeDerivative
根據使用者選擇的模型，回傳對應的導數值。

- 模型 1：回傳 -a * x，其中 a 代表 k。
- 模型 2：回傳 a * x * (1 - x / b)，其中 a 代表 r，b 代表 K。

2. simulate
負責執行 Euler 迭代：

- 初始值為 x0。
- 每一步使用公式 x = x + dt * dxdt 更新數值。
- 列印每一步的 step、time 與 x。
- 最後輸出最終結果。

3. main
負責讀取使用者輸入、檢查參數是否合法，並呼叫 simulate 進行模擬。

## 使用的數學模型

### 1. 線性模型
適合描述隨時間遞減的現象。

```text
dx/dt = -k*x
```

### 2. 非線性 Logistic 模型
適合描述具有成長上限的系統。

```text
dx/dt = r*x*(1 - x/K)
```

## 輸入參數說明
程式執行時會要求輸入以下資料：

- 模型編號：1 或 2
- 初始值 x(0)
- 時間步長 dt
- 模擬步數 steps

若選擇模型 1，另外需要輸入：

- k

若選擇模型 2，另外需要輸入：

- r
- K

## 輸入驗證規則
程式目前包含以下檢查：

- 模型只能輸入 1 或 2。
- dt 必須大於 0。
- steps 不能小於 0。
- Logistic 模型中的 K 不能為 0。
- 若輸入非數字資料，會顯示輸入格式錯誤訊息。

## 程式特色
- 以方法拆分導數計算與模擬流程，結構清楚。
